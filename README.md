# Custom Probe-Based Spatial Transcriptomics Enables Microbiome Detection in FFPE Colorectal Cancer Tissue

This repository contains the probe design pipeline supporting Aburajab et al., City of Hope.

## Overview

The pipeline is designed to create custom 16S rRNA split-ligation probe panels targeting microbiome species for use with the 10x Genomics Visium CytAssist platform. It takes FASTA sequences for each target species, generates consensus sequences via multiple sequence alignment, and designs probe pairs that pass QC filters derived from the 10x Genomics probe design specification.

## Dependencies

A conda environment file is provided to reproduce the exact environment used:

​```bash

​```

## Input File Formats

### accessions.tsv
Tab-separated file mapping species to NCBI accession numbers. Users should compile their own list based on target species of interest:
​
```
Fusobacterium nucleatum    NR_074412.1 NR_113198.1 NR_117287.1
Bacteroides fragilis    NR_112936.1 NR_074784.2 NR_112141.1 NR_119164.1
​```

### selected_probes.tsv
Tab-separated file created by the user after reviewing BLAST results. One row per selected probe:

```
original_probe_id    new_probe_id    gene_name    gene_symbol    composite_group
Fusobacterium_nucleatum_consensus_probe1    Fusobacterium_probe1    Fusobacterium    FUSO    1
Bacteroides_fragilis_consensus_probe1    Bacteroides_probe1    Bacteroides    BACTO    1
​```

## Downloading the 16S rRNA BLAST Database

The NCBI 16S rRNA RefSeq database is required for Step 3. To download:

​```bash
cd 16S_refseq/
update_blastdb.pl --decompress 16S_ribosomal_RNA
​```

`update_blastdb.pl` is included with NCBI BLAST+, which is installed as part of the conda environment.

## Downloading NCBI FASTAs

Accession numbers for target species should be compiled by the user and listed in `scripts/accessions.txt`. To download:

​```bash
cd scripts/
bash fasta_download.sh
​```

This requires NCBI Entrez Direct (`efetch`), installed as part of the conda environment.

## Usage

### Pipeline 1: Probe Design and BLAST

​```bash
WD_SEQ="/path/to/visium_microbiome" source scripts/run_probe_pipeline.sh
​```

**Step 1** — For each species in `accessions.tsv`, combines all accession FASTAs, aligns them with MAFFT, and generates a consensus sequence by taking the most common non-gap base at each position.

**Step 2** — Designs split-ligation probe pairs from consensus FASTAs. Each probe pair consists of a left half and right half, each 25 bp, tiling the reverse complement of the consensus sequence. Probes passing all QC filters are written to per-species TSVs, then aggregated into a single Excel file and FASTA file per genus.

**Step 3** — Combines all probe TSVs into a single FASTA and runs BLAST against the NCBI 16S rRNA database. Filtered results (percent identity ≥ 90%, query coverage ≥ 90%) are written to an Excel file with one sheet per genus for manual review.

### Pipeline 2: Probe Selection, QC, and Reference Generation

After reviewing BLAST results, populate `data/selected_probes.tsv` with the probes you want to include.

​```bash
WD_SEQ="/path/to/visium_microbiome" source scripts/run_qc_pipeline.sh
​```

**Step 4** — Looks up selected probes, renames them, runs secondary QC (melting temperature, hairpin, homodimer, heterodimer, junction quality, near-duplicate flagging), and generates a probe ordering file with appended handle sequences.

**Step 5** — Generates FASTA, GTF, and metadata files from the final probe set. These files are intended to be appended to an existing Visium CytAssist reference by the user — this step does not produce a complete reference on its own.
